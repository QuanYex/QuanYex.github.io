web安全入门

发表于 2020-02-03 | 分类于 学习

web流程：打开浏览器，浏览器向服务器发送请求，服务器与数据库进行交互，服务器向浏览器返回请求结果，浏览器显示。

浏览器工作原理：输入网址，浏览器通过域名获取到web服务器的ip地址（DNS解析），收到对应的ip地址后，想web服务器发送http请求，服务器返回http响应。

url：

作用：定位服务器的资源，充当于web服务器收货地址

格式：schema://host[:port#]/path/…/ [?query-string] [#anchor]

​ schema是底层协议（例如http，https，ftp）

​ host是服务器的域名或者ip地址

​ port是服务器端口，htto默认端口是80（可省略），其他端口需要指明

​ /path/…/是访问资源的路径

​ ？query-string是发送给http服务器的数据

​ #anchor 是锚点（通常表示页面的特定位置）

例如 http———-://dun.163.com——–/sj/test/test.jsp———?name=sviergn&x=treu———-#stuff

​ schema——————-host———————path————————-Query String—————–Anchor

http：

  1. 是什么超文本传输协议，是web通信时使用的协议，是web的基础。

类似于收快递 ：快递小哥——–方式：汽车———收件人地址：某某某———-快递公司：某某快递—–小区通行证———–收到（状态）+已签收（正文）

与之对应：——-浏览器———get、post请求等————-host———————user-agent——————cookie—————response：http200+正文。

  1. http报文：

    请求报文：请求行（request line）+头部（headers）+数据体（body）//get请求无数据体，数据体和头部之间有一行空行。

    响应报文：状态行+消息报头+响应正文 // 响应头部和正文之间也有空行。

    有些时候在请求报文中含有cookie字段。

  2. 请求方式 用途
    HEAD 与GET请求类似，不同在于服务器只返回http头部信息，没有页面内容。
    PUT 上传指定的URL的描述。
    DELETE 删除指定资源。
    OPTIONS 返回服务器支持的HTTP方法。

  3. http请求——referer：

    http请求字段含义拓展：http协议常用字段、HTTP协议头字段详解大全

    作用：告知服务器该请求的来源（浏览器自动加上）

    可以统计流量：CNZZ、百度统计 也可以判断来源合法性：防止盗链、防止CSRF漏洞。

  4. http响应：

    例如HTTP/1.1 302 Moved Temporarily （跳转的状态码（301/302））

    状态码拓展：http状态码

    set-cookie 服务端要求浏览器设置cookie

    location： 表示跳转到的地址

计算机网络概述

基本概念

  1. 计算机网络视通信技术与计算机技术紧密结合的产物，计算机网络就是一种通信网络

  2. 计算机网络是互连（互联互通 ——通过通信链路）的、自治（无主从关系）的计算机集合。

    注意：当距离很大时通过交换网络互联主机，交换节点（路由器或交换机）

  1. internet是全球最大的互联网络,是数以百万计的互连的计算设备集合：

    主机（host）=端系统（end system）

    通信链路：光纤、卫星、铜缆、无线电……

    分组交换：转发分组（数据包） 路由器（routers）和交换机（switches）

    问题：仅有硬件（主机、链路、路由器……）internet能否顺畅运行？能保证数据有序交付吗？

    ​ 不能，需要协议。

  2. 硬件是计算机网络的基础，计算机网络数据交换必须遵守规则，如同交通系统一样，这些规则就是协议。

  3. 网络协议（network protocol）为进行网络中的数据交换而建立的规则、标准或约定。规定了通信实体之间所交换的消息的格式、意义、顺序以及针对收到信息或发生的事件采取的“动作”

  4. 协议三要素：

    语法：结构或格式、信号电平 （怎么做）

    语义：需要发出何种控制信息、完成何种动作以及做出何种响 应、差错控制 （做什么）

    时序：时间顺序

  5. 计算机网络结构：

    1. 网络边缘：主机、网络应用
    2. 接入网络，物理介质：有线或无线通信链路
    3. 网络核心（核心网络）：互联的路由器、网络之网络。

  6. 网络边缘：

    主机：位于网络边缘、运行网络应用程序

    客户/服务器（client/sever）应用模型：客户发送请求，接收服务器响应

    对等（peer-peer，p2p）应用模型：无（或不依赖）专用服务器、通信在对等实体之间直接进行。如：QQ，Skype

  7. 接入网络：

    如何将网络边缘介入核心网？

    A：接入网络：住宅（家庭0、机构（学校、企业）、移动接入网络

    ​ DSL：数字用户线路（数据通信通过DSL电话线接入Internet）

    ​ 电缆网络： 和DSL相同采用频分多路复用技术。

    ​ HFC：混合光纤同轴电缆。共享电缆头端

    ​ 无线接入网络：无线局域网、广域无线接入（移动通信）

  8. 网络核心：

    关键功能：路由+转发。

    ​ 路由：确定分组从源到目的的传输路径（路由算法生成本地转发表）

    ​ 转发：将分组从路由器的输入端口交换至正确的输出端口。通过数据交换实现网络核心从源主机到达目的主机。

    N²链路问题 主机数为n 没两个主机能连接需要n(n-1)/2，技术很难实现，所以引入了交换设备，为了适应不同的网络规模，所以将交换设备连接成网络。

    应用层

    邮件传输协议——SMTP

    SMTP是一种提供可靠且有效的电子邮件传输的协议。SMTP是建立在FTP文件传输服务上的一种邮件服务，主要用于系统之间的邮件信息传递，并提供有关来信的通知。SMTP使用TCP端口25。测试

    SMTP交换过程中服务器端发出的“250 0K”含义是一切都好。与使用其他协议一样，程序只读缩写命令和每行开头的三个数字，其余文本是用于帮助用户调试邮件软件。在命令成功时，服务器返回代码250，如果失败则返回代码550（命令无法识别）、451（处理时出错）、452（存储空间不够）、421（服务器不可用）等，354则表示开始信息输入。

    SMTP的局限性表现在只能发送ASCII码格式的报文，不支持中文、法文、德文等，它也不支持语音、视频的数据。通过MIME协议，对SMTP补充。MIME使用网络虚拟终端(NVT)标准，允许非ASCII码数据通过SMTP传输。

    邮件访问协议——POP3、IMAP

    POP（post office protocol）规定了个人计算机连接到Internet的邮件服务器和下载电子邮件的协议。

    IMAP（Internet Mail Access Protocol）是Internet邮件访问协议，邮件客户端可以通过这种协议获取邮件服务器上面的邮件信息、邮件下载、发送等。

    区别：

    1. IMAP不用对服务器上面的邮件进行全部下载（根据实际需要进行下载），可以通过邮件客户端对邮件进行操作。

      POP3允许下载邮件服务器上面的邮件（下载所有未阅读的电子邮件），当邮件从服务器上面发送到电脑上面，同时删除邮件服务器上面的邮件（目前很多邮件服务器都支持“下载邮件，不删除邮件，或发出提醒”）。

    2. IMAP提供WebMail与邮件客户端进行双向通信，客户端上面的操作（如：已读、删除、移动邮件等）将会反馈到邮件服务器上面，在WebMail上面操作邮件时也会反馈到邮件客户端上面（即：在邮箱客户端、浏览器上面登录邮箱查看的邮件及状态是一致的）。

      POP3在客户端上面进行操作（如：已读、删除、移动邮件等）将不会反馈到邮件服务器上面，如：移动客户端上面的邮件到其他文件夹时，邮件服务器上面不会同时移动这些邮件。

    3. IMAP提供更加便捷、稳定、可靠的使用体验，通过WebMail与客户端进行双向通信可以避免邮件丢失。

      POP3更容易丢失邮件，也不能多次下载相同的邮件（目前的很多POP3邮件服务器已经支持保留原邮件，支持多次下载）。

    4. IMAP提供了从多台设备（如：手机、电脑、平板电脑等）访问邮件，便于随时随地查阅邮件。IMAP提供的摘要浏览功能便于查阅邮件的到达时间、主题、发件人、大小等信息，然后再决定是否需要下载、打开。

    DNS协议

    DNS（Domain Name System）是互联网的一项服务。它作为将域名和IP地址相互映射的一个分布式数据库，能够使人更方便地访问互联网。DNS使用TCP和UDP端口53。当前，对于每一级域名长度的限制是63个字符，域名总长度则不能超过253个字符。DNS协议是用来将域名转换为IP地址（也可以将IP地址转换为相应的域名地址）。

    DNS是多层命名服务器构成的分布式数据库，一般用来

    • 域名向IP地址的翻译
    • 主机别名
    • 邮件服务器别名
    • 负载均衡

    当主机进行DNS查询时，查询被发送到本地域名服务器（每个ISP（运营商）都有一个本地域名服务器）本地域名服务器作用：

    • 作为代理，将查询发给（层级式）域名解析服务器系统
    • 在缓存中查询（缓存一段时间会失效，一般缓存的事根域名服务器）

    DNS格式：资源记录（RR， resource records）

    格式为： name  value  type  ttl
    • type=A name为主机域名 value为ip地址
    • type=NS name为域名（edu.cn） value为该域权威域名解析服务器的主机域名
    • type=CNAME name为某一真实域名的别名 value为真实域名
    • type=MX value时与name相对应的邮件服务器

    DNS查询有两种方式：

    • 递归查询
    • 迭代查询

    从请求主机到本地DNS服务器的查询是递归的，其余的查询是迭代的

    DNS负载均衡

    DNS负载均衡技术的实现原理是在DNS服务器中为同一个主机名配置多个IP地址，在应答DNS查询时，DNS服务器对每个查询将以DNS文件中主机记录的IP地址按顺序返回不同的解析结果，将客户端的访问引导到不同的机器上去，使得不同的客户端访问不同的服务器，从而达到负载均衡的目的。

    DNS用的是TCP协议还是UDP协议

    DNS占用53号端口，同时使用TCP和UDP协议。那么DNS在什么情况下使用这两种协议？

    DNS在区域传输的时候使用TCP协议，其他时候使用UDP协议。

    DNS区域传输的时候使用TCP协议：

    1.辅域名服务器会定时（一般3小时）向主域名服务器进行查询以便了解数据是否有变动。如有变动，会执行一次区域传送，进行数据同步。区域传送使用TCP而不是UDP，因为数据同步传送的数据量比一个请求应答的数据量要多得多。

    2.TCP是一种可靠连接，保证了数据的准确性。

    域名解析时使用UDP协议：

    客户端向DNS服务器查询域名，一般返回的内容都不超过512字节，用UDP传输即可。不用经过三次握手，这样DNS服务器负载更低，响应更快。理论上说，客户端也可以指定向DNS服务器查询时用TCP，但事实上，很多DNS服务器进行配置的时候，仅支持UDP查询包。
